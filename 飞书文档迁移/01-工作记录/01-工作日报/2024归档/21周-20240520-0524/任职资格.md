任职资格
# 技术攻关 = 已经补充攻关审核截图、以及飞帆的攻关认证相关材料
1、举证的事例要事先通过集团正式的技术攻关等级认定；
2、举证2个C级攻关事例。。需要举证攻关系统上评审通过的案例。举证的事例要事先通过集团正式的技术攻关等级认定
# 架构设计 =  已经补充V1.0架构设计、V1.1的架构评审邮件，以及说明2个案例的情况
1、举证评审结论的邮件。架构设计需要闭环材料，同时需要举证2个案例
# 详细设计 = 增加说明V1.1的详细设计：服务拆分、架构优化等，同时补充评审邮件
1、需要举证2个详细设计的案例
# 设计跟踪与指导 = 增加详细设计的的执行情况、补充向日葵集成举例情况
1、需要举证详细设计向团队传达，以及如何验证团队是按照你的详细设计完成代码实现的

# 系统编码 = 增加M3.0架构优化代码数量、代码优化后测试报告效果
1、需要举证你写了什么代码， 并举证这个代码的规范性和质量满足要求
# 测试和验证 = 增加融通M3.0的IM以及语音测试方案的复杂性
1、需举证场景的复杂性
# 预见与避免错误 = 补充提前四川中行的提前进行识别并且给出了方案、增加融通高峰期前的数据库数据量的问题
1、不是问题解决，而是要提前发现潜在的风险，而不是必然出现的问题。举证未体现以下两点
2.此类问题应该是实际发生之前被申请人识别出来，并采取了措施避免问题发生；
3.需同时举证识别出的技术问题和所制定的解决方案
# 线上问题分析和解决 = 补充了融通网络情况的复杂性部署情况问题的复杂度
1、生产环境问题复杂度（环境、业务量等）缺少
# 技术共享 = 补充使用innsetup进行二次封装而不是二次开发，给项目带来的效果带来的项目成本节约
1、需举证引入到自己项目中的可复用资产以及因此节省的成本；如举证贡献，需要举证使用方产生的效果及范围。至少举证2个案例。需要举证的是复用的模块、组件案例而非逻辑设计
# 技术创新 = 补充说明es的跨机器复制方案ccr
1、需要举证技术调研报告和决策影响证明
